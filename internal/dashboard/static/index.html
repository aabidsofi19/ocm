Metrics Explained
CSA — Configuration Surface Area

Spec definition: count of configurable parameters + environment variables + resource specifications + replica configurations.

This POC extracts (from Kubernetes YAML): container env entries, container ports, leaf keys under resources, presence of spec.replicas, plus a heuristic count of top-level keys under spec.

DD — Dependency Depth

Spec definition: longest path length (edge count) in the service dependency graph, where A → B means “A depends on B”.

This POC infers edges from env vars whose names end with _SERVICE or _SERVICE_HOST and whose values look like a service hostname.

Cycles are handled deterministically by condensing strongly-connected components into a DAG and computing the longest path there.

OCM — Composite Score

Normalization: per run cohort (all services in that run), (M - min) / (max - min). If max == min, normalized value is 0.

MVP weights: 0.5 for CSA and 0.5 for DD. Missing metrics are treated as 0.
